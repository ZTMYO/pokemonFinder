# Pokémon Finder

中文 | [English](README.en.md)

一个支持“**凭外貌找名字**”和“**凭名字看外貌**”的宝可梦检索小工具。

前端为纯静态页面，无需后端服务，所有计算都在浏览器本地完成。

![项目预览图](img.png)

## 功能

- **外貌 → 名字：按颜色匹配宝可梦主色**
  - 通过取色器 / hex 值添加最多 5 个颜色。
  - 点击 `开始匹配`，根据颜色组合从全图集中找出最接近的宝可梦。

- **名字 → 外貌：按名称 / 拼音搜索**
  - 顶部搜索框支持：中文名、英文名、拼音、拼音首字母。
  - 输入后按 `Enter`，在右侧显示第一条匹配到的宝可梦卡片。

- **筛选与展示**
  - 筛选：属性（最多 2 个）、世代、体型、Mega / G‑Max。
  - 展示：右侧卡片列表，可设置结果数量，每张卡片展示名称、编号、属性等信息，并附带主色条。

- **主题与布局**
  - 支持明暗主题切换。
  - 桌面端左右分栏；移动端自动改为上下布局并自适应列数。

## 使用方式（简要）

1. **选颜色匹配**
   - 点击 `添加颜色`，用取色器或输入 hex 值补充颜色。
   - 设好筛选条件后，点击 `开始匹配` 查看结果。

2. **按名字搜长相**
   - 在顶部搜索框输入中文 / 拼音 / 首字母 / 英文，按 `Enter`。
   - 右侧会仅显示第一条匹配结果；修改筛选控件会回到颜色匹配模式。

3. **调整结果**
   - 可修改匹配方式、结果数量、属性 / 世代 / 体型 / Mega / G‑Max 等筛选条件，结果会自动刷新。
   - 使用 `重置` 恢复默认设置并清空颜色行与筛选。

## 数据与扩展

- 颜色数据来自对官方宝可梦图像的预处理，每只宝可梦有若干代表主色的十六进制值。
- 若要替换成其他图片或数据源，只需保持 `pokemon_data.json` 字段结构一致即可。
- 宝可梦基础数据在本项目中做了整理与增强，原始数据参考：[42arch/pokemon-dataset-zh](https://github.com/42arch/pokemon-dataset-zh)。
